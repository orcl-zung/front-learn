第 3 课 · 阶段二：CSS 基础

Flexbox：一维布局的主力

导航栏、按钮组、表单标签和输入框并排、卡片排成一行 —— 这些"把东西排成一条线"的活， 全是 Flexbox 的主场。它是一维布局，类比 Swing 的 BoxLayout， 但对齐模型干净得多。这课后，你不再靠碰运气凑布局。

本课目标

学完你能做到：

  • 看到任何"一排 / 一列元素"的界面，知道该用 Flexbox，并能立刻指出主轴和交叉轴。
  • 用 display: flex 加几个容器属性，排出居中、两端对齐、自动换行等常见效果。
  • 解释 flex-grow / shrink / basis 三件套如何分配空间（类比 BoxLayout 的 glue / strut）。
  • 说清为什么 Flex 容器里的 margin 不折叠 —— 呼应第 2 课那个"历史怪癖"。

为什么这课是后台开发的看家本领

你过去粘 Vue 后台模块时，"按钮排一排""标签和输入框对齐""顶栏 logo 在左、菜单在右"几乎每页都有。 那时布局是改到大概齐就收手。Flexbox 给你一个稳定的内心模型：心里有主轴， 布局从"试出来的"变成"想出来的"。而且它正是后面 React/Vue 组件里最常用的排版手段。

一、Flexbox 是什么：一维布局（先给 Java 骨架）

"一维"的意思是：它一次主要管一个方向 —— 要么排成一行，要么排成一列。 要同时管行和列（像表格、像画廊），那是下一课 Grid 的活。

Flex 永远有两个角色：

  • flex 容器（父元素）：你给它设 display: flex，它就不再按默认的块级/流式排版，而是改用 Flex 规则摆它的直接子元素。
  • flex 项目（那些子元素）：被容器沿着一条轴排开的盒子。
类比 · display:flex ≈ 给容器换上 BoxLayout

Swing 里你写 new BoxLayout(panel, BoxLayout.X_AXIS)，panel 就变成 "把子组件从左到右排成一行"；Y_AXIS 则是从上到下排成一列。CSS 里这一步是给容器写 display: flex，方向由 flex-direction 决定（row ≈ X_AXIS， column ≈ Y_AXIS）。差别在下一步：BoxLayout 里你要手动塞 Box.createHorizontalGlue()（可伸缩的占位）去把元素推开、用 setAlignmentX/Y 调对齐；Flex 把这些做成了容器上的声明式属性（justify-content / align-items）， 默认值就很合理，不用一个个组件去调。

二、唯一要先钉死的模型：两条轴

Flex 九成的困惑，都来自没分清两条轴。先把这个模型刻进脑子：

  • 主轴（main axis）：flex-direction 指定的方向。row → 水平；column → 垂直。项目沿着主轴排开。
  • 交叉轴（cross axis）：与主轴垂直的那条。

为什么这么重要？因为两套对齐属性各管一条轴：justify-content 管主轴， align-items 管交叉轴。下面这张图是 flex-direction: row 的情形 —— 主轴水平、交叉轴垂直：

四、项目怎么分空间：flex-grow / shrink / basis

上面是"项目保持自己大小、容器管对齐"。但很多时候你要的是项目主动填满空间： 侧边栏固定宽、主内容区吃掉剩下所有的宽。这就得给项目设属性：

  • flex-basis：项目的初始大小（沿主轴）。类比 Java 组件的 preferredSize。不写就按内容宽。
  • flex-grow：容器有剩余空间时，按这个比例分给项目。0 = 不抢（默认）。
  • flex-shrink：空间不够时，按这个比例让项目收缩。1 = 等比缩（默认）。

类比 · flex-grow ≈ BoxLayout 的 glue

Box.createHorizontalGlue() 就是一个看不见、但会吃掉所有多出来空间的组件 —— 你塞一个在两个按钮中间，按钮就被推到两端。Flex 把这个机制做进了每个项目：给项目设 flex-grow: 1，它自己就变成那根"胶"，无需额外插占位组件。要"两块均分"，就两块都设 flex: 1；要 1:2，就 flex: 1 和 flex: 2。这就是上面滑块演示的原理。

五、单个项目特立独行：align-self 与 order

  • align-self：容器统一了 align-items，但某一个项目想例外（比如其他都居中、就它顶部对齐），就给它单独写 align-self。取值和 align-items 一样。
  • order：改项目的视觉顺序（默认都 0，越小的排越前）。慎用：它只动视觉、不动 DOM 顺序，屏幕阅读器和键盘 Tab 仍按 DOM 走 —— 视觉和无障碍顺序对不齐会坑用户。能靠改 HTML 顺序解决的，就别用 order。

六、三个必背配方

这三段代码覆盖了你在博客里最常遇到的排版。把它们和 Java 的做法对比 —— 你会发现声明式的 CSS 短得不像话。

配方 1：把一个元素居中（经典中的经典）

.cover { display: flex; justify-content: center; align-items: center; }

主轴居中 + 交叉轴居中 = 完全居中。在 Java 里你得算坐标 / 用 GridBagLayout 配 anchor=CENTER，这里一行搞定。博客封面图上的标题文字居中，就这么写。

配方 2：导航栏（logo 左、菜单右）

.nav { display: flex; justify-content: space-between; align-items: center; }
类比 · 这就是 BorderLayout 的 NORTH 行

Java 里顶栏常用 BorderLayout：WEST 放 logo、EAST 放菜单按钮。 Flex 的 justify-content: space-between 达到一模一样的效果，而且一行属性、不用嵌套两个区域。 align-items: center 再保证 logo 和菜单垂直居中对齐，不会一个高一个低。

配方 3：标签 chips 自动换行 + 等间距

.tags { display: flex; flex-wrap: wrap; gap: .5rem; }

flex-wrap: wrap 让 chips 放不下时自动折行；gap 统一间距。 注意这里完全没用 margin —— 见下一节。

七、呼应第 2 课：Flex 里 margin 不折叠

第 2 课讲过一个历史怪癖：两个垂直相邻的块元素，上下 margin 会折叠（取较大者）。 现在给你一个好消息：

一旦父容器是 Flex（或下节课的 Grid），子项目的 margin 不再折叠 —— 垂直方向的 margin 也会正常相加。

所以现代布局里你基本碰不到 margin 折叠。更进一步：项目之间的间距，推荐用容器的 gap， 而不是给每个项目写 margin。原因有二：① gap 只在项目之间生效，不会在容器边缘多出一条 （用 margin 你常常得写"去掉最后一个的右边距"这种补丁）；② 语义清晰 —— 间距是容器的职责，不是项目的。 这就是为什么上面配方 3 用 gap 而非 margin。

博客落地：导航栏 + 标签条 + 卡片行

把今天钉到你的博客项目：

  • 顶部导航栏：站名在左、导航菜单在右 → 容器 display:flex; justify-content:space-between; align-items:center（配方 2）。
  • 文章的标签条：一排 chips、放不下折行 → display:flex; flex-wrap:wrap; gap:.5rem（配方 3）。
  • 文章列表：卡片排成一行、等宽填满 → 容器 display:flex; gap:1rem，每张卡片 flex:1。整齐排列用 Flex 完全够；等以后要做"严格网格 + 跨行跨列"，再请出下一课的 Grid。

八、自检：两轴你挂对了吗？

第 1 题 · 主轴方向
flex-direction: row 下，justify-content 管的是哪个方向的对齐？

  • 水平方向（主轴）
  • 垂直方向（交叉轴）
  • 两个方向都管
  • 取决于 align-items

justify-content 永远管主轴。row 的主轴是水平的，所以它管水平对齐。 记住口诀：主轴 = 排开项目的方向；justify-content 跟着主轴走，align-items 跟着交叉轴走。

第 2 题 · 居中配方
把一个元素在父容器里完全居中，需要哪两个属性（容器上）？

  • justify-content: center + flex-direction: column
  • align-items: center + flex-wrap: nowrap
  • justify-content: center + align-items: center
  • text-align: center + vertical-align: middle

主轴居中（justify-content）+ 交叉轴居中（align-items）= 两个方向都居中 = 完全居中。 前提是父容器 display:flex。text-align 只管行内文字，做不到块级居中。

第 3 题 · flex-grow 比例
同一行三个项目，A flex-grow:1、B flex-grow:2、C flex-grow:1（basis 都为 0）。剩余空间怎么分？

  • A 30% / B 30% / C 30%（均分）
  • A 25% / B 50% / C 25%
  • A 50% / B 25% / C 25%
  • A 33% / B 66% / C 0%

grow 之和 = 1+2+1 = 4。各分得 grow/总和：A=1/4=25%，B=2/4=50%，C=1/4=25%。 flex-grow 是按比例分剩余空间，比例就是它的值。这正是上面滑块 demo 演的事。

第 4 题 · gap vs margin
Flex 项目之间的间距，为什么推荐用容器的 gap 而不是给每个项目写 margin？

  • gap 只在项目之间生效，不在容器边缘多出间距，也无需"去最后一个边距"的补丁
  • margin 在 Flex 里完全不生效
  • gap 能让项目自动换行，margin 不能
  • gap 比 margin 渲染更快

margin 在 Flex 里照样生效（只是不折叠），但用 margin 控间距常需写"去掉最后一项的边距"这种补丁， 且边缘也会留白。gap 只作用于项目之间，语义干净。换行由 flex-wrap 控制，与 gap/margin 无关。

第 5 题 · 导航栏配方
实现"logo 贴左、菜单贴右、两者垂直居中对齐"的导航栏，容器该用？

  • justify-content: center; align-items: center
  • justify-content: flex-start; align-items: stretch
  • justify-content: space-between; align-items: center
  • justify-content: space-around; align-items: flex-end

space-between 把首尾两个项目分别贴到起点和终点（logo 最左、菜单最右）， align-items: center 让它们在交叉轴（垂直方向）居中对齐。这就是博客顶栏的标准写法（配方 2）。
