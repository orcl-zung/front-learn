第 1 课 · 阶段一：定位与地图

前端世界地图：用 Java 的眼睛看一遍

把整个前端世界，挂到你已熟知的 Java/JVM/Spring 地图上。本课不写一行业务代码 —— 它要做的事更重要：拆掉「前端很神秘」的心理墙。

本课目标

学完你能做到：看到任何一个前端术语（组件、props、响应式、虚拟 DOM、事件循环……）， 不再发懵，而是立刻在脑子里挂到一个 Java 对应概念上，并知道它属于哪一层。这是后续一切的地基。

为什么从这开始

你过去能把后台界面「粘出来」，却不敢动底层，是因为缺这张地图。 框架（React/Vue）只是冰山一角；它下面压着 CSS、现代 JS、TypeScript、浏览器运行时和工具链。 不知道分层，就只能在最上层抄 —— 一旦需求偏离模板就卡死。我们自下而上打通它。

一、分层地图：浏览器 ≈ 前端的 JVM

一个网页跑起来，依赖下面这几层。从下往上看，每层都给上层提供能力，越往上越接近你写的业务代码：

React / Vue 框架层 — ≈ 一个声明式的 UI 框架（像 JavaFX/Swing，但描述式）
把「数据 → 视图」的同步自动化：你只声明 UI 是数据的函数，框架替你接好监听器。
TypeScript / 现代 JS（语言层） — ≈ Java 语言 + 静态类型
TS 是你的主场。现代 JS（ES2020+）提供 let/const、箭头函数、解构、模块、Promise/async。
CSS（样式层） — ≈ JavaFX 的 CSS / 声明式样式表
你的最大缺口。决定「长什么样」：颜色、布局、间距、响应式。
HTML → DOM（结构层） — ≈ JavaFX 的 Scene Graph / Swing 组件树
HTML 是源码；浏览器把它解析成一棵可被 JS 操作的对象树，叫 DOM。
浏览器运行时（底层） — ≈ JVM + 标准库 + 图形运行时
提供 JS 引擎、DOM、网络、存储、渲染，以及一个单线程的事件循环。
工具链：npm / Vite（工程层） — ≈ Maven/Gradle + javac
管依赖、转译 TS/JSX、热更新、打包。你写的源码要先被它处理，浏览器才能跑。

二、两个「杀手级」类比（务必记住）

类比 1 · 事件循环 ≈ Swing 的 EDT

浏览器的 JS 是单线程的：所有代码、事件回调、UI 重绘，都排在一个队列里轮流执行。 这跟 Swing 的事件派发线程（EDT）一模一样 —— 你不会在 EDT 上做耗时操作，否则界面卡死； 前端同理，耗时任务要用异步（Promise / async）。理解了这一点，JS 的异步就不再反直觉。

类比 2 · 响应式 ≈ PropertyChangeSupport / ChangeListener

在 Java 桌面端，你要手动给字段加监听器：字段变了 → 通知 UI → 重绘。 React/Vue 把这件事自动化了 —— 你只声明「这块 UI 是这个数据的函数」，数据一变，框架自动刷新视图。 响应式，就是「数据驱动视图」的自动化监听机制。这是整个框架层的灵魂。

三、你的家底盘点

✅ 你已经会的（优势）

  • 强静态类型思维 → TS 几乎白送
  • 工程化心智（Maven/构建/依赖）→ npm/Vite 迅速迁移
  • 组件/分层/接口设计 → 组件化 UI 思想同源
  • 事件/监听/异步 → 响应式与事件循环有 Java 根基

⬜ 要补的（缺口）

  • CSS（几乎零）→ 系统补：盒模型/Flex/Grid/响应式
  • 现代 JS 语法与 DOM API → 能读不能写
  • 浏览器运行时模型 → 事件循环、异步
  • 从「粘」到「懂」→ 每段代码都要能解释为什么

四、学习路线图

  1. 前端世界地图（本课，建立坐标系）
  2. CSS 基础：盒模型 → Flexbox → Grid → 响应式（系统补缺口）
  3. 现代 JavaScript：语法、DOM、异步（事件循环 / Promise / async）
  4. TypeScript：你的主场，快速拿下（重点：结构类型 vs 名义类型）
  5. React 深入：组件、props、state、Hook、副作用、表单、路由
  6. Vue3 对照：组合式 API，复现同一功能，讲清差异
  7. 落地：用你的项目，从零独立交付一个前端

五、自检：这些 Java 类比你挂对了吗？

第 1 题 · 工具链
前端的 npm + package.json，最像 Java 世界的？

  • Maven 与 pom.xml
  • JUnit 与测试类
  • Git 与版本库
  • Lombok 与注解

npm 管依赖、package.json 声明依赖与脚本，正是 Maven + pom.xml 的角色。

第 2 题 · 运行时
浏览器执行 JS 的「事件循环」，最像？

  • Swing 的事件派发线程
  • 垃圾回收线程
  • Spring 的调度器
  • Tomcat 的线程池

单线程轮询任务队列、派发回调与重绘 —— 与 Swing EDT 同构。这也是 JS 异步的设计根源。

第 3 题 · 组件
一个组件接收的 props，最像 Java 里的？

  • 构造方法的入参
  • 类的静态常量
  • 私有的实例字段
  • 抽象的方法

props 是父传子的单向、只读数据，相当于构造入参。私有的可变字段对应的是 state，不是 props。

第 4 题 · 框架灵魂
框架的「响应式 reactivity」，最像 Java 里的？

  • 属性变更监听器
  • synchronized 同步锁
  • 序列化 Serializable
  • 反射 Reflection

响应式 = 数据变了自动通知视图刷新，即 PropertyChangeSupport / ChangeListener 的自动化版本。
