第 2 课 · 阶段二：CSS 基础

盒模型：每个元素的四层矩形

浏览器把每个元素都画成一个矩形盒子，盒子由四层套娃组成。 这是 CSS 的第 0 公理 —— 后面所有布局（Flex、Grid）都建立在你能否在脑子里"看见"这个盒子。

本课目标

学完你能做到：

  • 打开浏览器 DevTools，看一眼盒模型彩图，立刻说出每层是什么、多少像素。
  • 能解释"为什么这个元素比我设的 width 要宽"。
  • 会用一行 box-sizing 杜绝 99% 的尺寸溢出坑。

为什么从盒模型开始

你过去粘后台界面时遇到的间距对不齐、内容溢出、按钮宽窄不一，根因几乎都是没建立盒模型心智。 它是 CSS 的地基。好消息：Java 桌面端那个"组件占多大地方"的模型，就是它的近亲 —— 你比零基础快得多。

一、盒模型：每个元素都是四层套娃

浏览器渲染时，把每个元素（哪怕一段 <p> 文字）当成一个矩形盒子， 由外到内四层：margin（外边距）→ border（边框）→ padding（内边距）→ content（内容）。下面这张图就是这四层 —— 鼠标移到每一层上看看：

margin · 外边距
border · 边框
padding · 内边距
content · 文章标题 + 摘要

背景色（background）只铺到 content + padding；margin 是完全透明的，它只是"占位"。

层CSS 属性含义Java 对应
margin外边距盒子外部的空白，推开别的元素；背景涂不到布局管理器的 gap（FlowLayout 的 hgap/vgap）
border边框盒子的边框线，有粗细 / 样式 / 颜色LineBorder / BorderFactory.createLineBorder()
padding内边距边框到内容的留白（呼吸空间）EmptyBorder / Insets 的内填充
content内容真正放东西的区域（文字 / 子元素）组件 preferredSize 的内容区
类比 · 盒模型 ≈ 一个 Swing 组件的占地

把一个 JButton 摆出来：按钮文字是 content；给它套个 EmptyBorder 就是 padding（内填充）；再套个 LineBorder 就是 border；把它丢进 FlowLayout，组件之间由 hgap/vgap 拉开的距离就是 margin。这个组件最终"占多大地方"，就是这四层之和。

二、padding vs margin：到底用哪个

这是新手最常卡的选择题。记住一句口诀就够了：

padding 是"我给自己留白"，margin 是"我推开别人"。
  • padding = 元素自己的内留白（边框 ↔ 内容之间）。
    博客场景：文章卡片里，文字到卡片边的呼吸距离 → padding。
  • margin = 元素和别的元素之间的距离。
    博客场景：列表里两张文章卡片之间的间隔 → margin。

还有个判别法：元素有背景色 / 边框时，背景只覆盖 content + padding。 所以你想要"带颜色的留白"用 padding，想要"透明的间隔"用 margin。

三、最大的坑：box-sizing

这是 CSS 最反直觉的默认值，也是你过去踩过无数坑的源头。

默认 box-sizing: content-box：你写的 width 只管 content。 一旦你加了 padding / border，盒子实际占地会比 width 大。

而 box-sizing: border-box：width 包含 content + padding + border， 盒子实际占地就是你写的 width。

点下面按钮，亲眼看同一行代码在两种模式下盒子尺寸怎么变：

content

看得出来为什么几乎所有项目都在样式最开头写这一行了吗：

*, *::before, *::after {
  box-sizing: border-box;
}
类比 · border-box ≈ 你设的 width 就是组件占地总宽

在 Java 里，setPreferredSize(200, 40) 就是这个组件的总占地 —— 你不会预期给它加个 EmptyBorder 之后还撑出 200 以外。CSS 的 border-box 就是让 width 表现得像 Java 的这个直觉；而默认的 content-box 反人类，所以全行业都重置成 border-box。把这行写进每个项目的起点，你就和 Java 的直觉对齐了。

四、margin 折叠：一个历史怪癖

两个垂直相邻的块元素，它们的上 / 下 margin 不相加，而是取较大者（塌陷合并）。

例：卡片 A margin-bottom: 24px，紧跟着的卡片 B margin-top: 16px → 两者实际间距是 24px，不是 40px。

Java 没这个对应物

这是 CSS 的历史包袱，纯粹记住即可。好消息：一旦进入 Flex / Grid 布局（后面几课）， margin 不再折叠。所以这个坑主要影响传统的块级文档流，现代布局里基本遇不到。

博客落地：一张文章卡片

把今天的概念钉到你的项目上。一张博客文章卡片：

  • content = 文章标题 + 摘要 + 发布日期等元信息。
  • padding = 卡片内文字到卡片边的呼吸空间（比如 16px）。
  • border = 卡片描边（1px），或干脆无边框改用阴影。
  • margin = 列表里卡片彼此的间距（比如 16px）。

动手：打开本课页面，按 F12，选中上面那个彩色盒子的最外层， 在 Computed 里把它的 padding 改大，亲眼看内层怎么被挤 —— 这就是你以后调样式的日常。

六、自检：四层你挂对了吗？

第 1 题 · padding 的 Java 对应
CSS 的 padding，最像 Java 里的？

  • FlowLayout 的 hgap
  • EmptyBorder / Insets 内填充
  • LineBorder 的厚度
  • setPreferredSize()

padding 是边框到内容的内留白，正是 EmptyBorder 制造的效果、Insets 描述的量。 hgap 是组件之间的间距（margin 的角色），LineBorder 是边框（border 的角色）。

第 2 题 · 默认 content-box
默认盒模型下，width: 300px + padding: 20px（左右各） + border: 5px（左右各），盒子实际占地宽？

  • 300px
  • 330px
  • 350px
  • 340px

content-box 下 width 只算 content。padding 水平共 40，border 水平共 10， 合计 300 + 40 + 10 = 350px。这就是"盒子比预期宽"的来源。

第 3 题 · 切到 border-box
同样设置，但加了 box-sizing: border-box，盒子宽？其中 content 宽？

  • 盒子 300px，content 250px
  • 盒子 350px，content 300px
  • 盒子 300px，content 300px
  • 盒子 350px，content 250px

border-box 下 width:300px 就是总占地。content 被挤成 300 − 40(padding) − 10(border) = 250px。 这正是 Java setPreferredSize 的直觉。

第 4 题 · 背景铺到哪
给一个元素设了 background，颜色会覆盖到哪一层？

  • 只有 content
  • content + padding（border 内侧以内）
  • content + padding + margin
  • 只有 padding

背景铺到 border 内边缘以内，即 content + padding。margin 永远透明。 所以"带颜色的留白"用 padding，"透明的间隔"用 margin。
