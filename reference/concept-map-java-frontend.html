Java ↔ 前端 概念对照表

把每一个陌生前端概念，钉到一个你已熟知的 Java/JVM/Spring 概念上。 带 ★ 的是「关键且极易出错」的类比 —— 那里两者并不完全相同，注意差异。

使用方法：遇到新词先来这里查它的 Java 骨架，再去 术语表 看精确定义， 最后回到当节课做练习。这份表会随课程推进不断扩充。

一、工具链 / 工程

前端概念Java 类比说明 / 差异
npm / pnpm / yarnMaven / Gradle包管理器。npm 是事实标准。
package.jsonpom.xml / build.gradle声明依赖与脚本命令。
node_modules/~/.m2 本地仓库区别：依赖默认装进项目目录内，体积大。
Vite / 打包器(bundler)javac + jar（但能力更多）转译 TS/JSX、压缩、tree-shaking、开发热更新。
ES Module import/exportJava import / package模块的导入导出机制。
nodeJVMJS 运行时；但浏览器自带 JS 运行时，node 主要服务构建与 SSR。

二、语言层：TypeScript 是你的主场

前端概念Java 类比说明 / 差异
TypeScriptJava 的静态类型你最大的优势。TS 是 JS 的超集，类型在编译期擦除。
interface / typeJava interface★ TS 是结构类型(duck typing)，Java 是名义类型。形状对了就算匹配，不需要 implements。
泛型 <T>Java 泛型 <T>概念几乎一致。
联合类型 A | B无直接对应（Java 21 前无）★ 一个值可以是 A 或 B。Java 21 的 sealed + pattern 近似。
可选 ?: / null | undefined@Nullable / OptionalTS 用类型表达「可能没有」。
装饰器 decorator注解 @Annotation概念相同；TS 装饰器是运行时可调用的函数。

三、运行时：浏览器 ≈ 前端的 JVM

前端概念Java 类比说明 / 差异
浏览器JVM + 标准库 + 图形运行时提供 JS 运行时、DOM、网络、存储、渲染。
DOMJavaFX Scene Graph / Swing 组件树一棵浏览器维护的对象树，对应页面结构。
事件循环 event loop ★Swing 的 EDT（事件派发线程）单线程。所有 UI 更新与回调排在这一个队列里。这是理解 JS 异步的钥匙。
Promise / async-awaitCompletableFuture异步结果的占位；await ≈ 阻塞式写法的非阻塞等价物。
回调 callbackListener / 回调接口「事情发生时请调用我」。
setTimeout / 宏任务SwingUtilities.invokeLater / ScheduledExecutor把任务排进事件循环。

四、样式：CSS（你目前的最大缺口）

前端概念Java 类比说明 / 差异
CSSJavaFX CSS / FXML 样式表声明式地描述外观，与结构分离。
盒模型 box model组件的 insets / 边距每个元素 = margin + border + padding + content 四层矩形。
Flexbox ★BoxLayout / 水平垂直排列的布局管理器一维排列（行或列）的首选。
GridGridLayout / GridBagLayout二维网格布局的首选。
选择器 selector无（声明式，按规则匹配）决定「哪些元素被这条规则影响」。
媒体查询 @media无（运行时判断视口尺寸）响应式：不同屏幕宽度用不同样式。

五、框架：React / Vue（它们管理 DOM + 状态）

前端概念Java 类比说明 / 差异
组件 component一个可复用的类 / BeanUI 的基本积木。React 里是函数或类；Vue 里是 .vue 单文件。
props ★构造方法入参 / setter（只读）父→子的单向数据。props 是只读的，别在子组件里改它。
state 状态可变的实例字段组件自己持有、可改的数据。改了会触发重新渲染。
响应式 reactivity ★PropertyChangeSupport / JavaFX 的 ChangeListener核心思想：数据变了，视图自动更新。Vue 自动追踪；React 靠 setState/触发。
渲染 renderpaintComponent / 重绘视图根据当前 state 产出 UI 描述。
虚拟 DOM virtual DOM脏标记 dirty checking + 批量 flush先在内存里算 diff，再最小化操作真实 DOM。
生命周期钩子@PostConstruct / @PreDestroy / Servlet init,destroy组件创建、更新、销毁时挂载逻辑。
Hook (React) ★把「类的实例能力」抽成可复用函数★ 名字相同但与「生命周期钩子」是两回事。use 开头的函数，在函数组件里复用状态等逻辑。
组合式 API (Vue)同上理念（函数式组织逻辑）Vue 3 对应 React Hook 思路的方案。
SPA 路由 router客户端内部自己切视图（不刷新页面）类似一个带 @RequestMapping 的桌面程序在本地切屏。
状态管理 Pinia / Redux应用级单例 Service / 全局上下文跨组件共享的、可预测的状态仓库。

一句话心法

框架做的事情，本质是：把「数据 → 视图」的同步自动化。 在 Java 桌面端你要手写 Listener 把字段变化刷到 UI；React/Vue 让你只声明「UI 是数据的函数」，框架替你接 Listener。
