# Mission: 从 Java 高级开发转型全栈 — 真正掌握 React / Vue

## Why
用户是 Java 高级开发，长期目标是从后端扩展为能独立交付前后端的全栈。
当前痛点：能用 Vue2/Vue3 把已有后台模块「粘贴复制」拼出表单、弹窗，但底层
(CSS 几乎为零、现代 JS/TS 能读不能写、对浏览器运行时与响应式机制没有心智模型)
是黑盒，一旦需求偏离模板就卡住。要打破这个黑盒，建立能独立设计、调试、扩展前端的能力。

## Success looks like
- 看到任何前端术语，能立刻在脑子里挂到一个 Java/JVM 对应概念上（不再发懵）。
- 能徒手写出规范的 CSS：盒模型、Flexbox、Grid、定位、响应式断点，不再靠抄样式。
- 能读懂并写出 ES2020+ 的现代 JS 与带类型的 TypeScript（TS 是其主场，应快速拿下）。
- 用 React（优先）从零搭出一个带状态、组件、路由、表单校验的真实功能 —— 不是粘贴。
- 能在 Vue3 组合式 API 中复现同一功能，说出它与 React 的关键差异。
- 把上述能力落到真实项目：**个人博客系统**（文章展示、动效、SEO、better-auth 认证、
  RBAC 动态权限、全文搜索、客户端可用的后端 API），独立交付前端。

## 项目（落地目标）
全部课程示例改用博客域（文章卡片、评论表单、文章排版等），不用泛泛的 todo demo。
功能与对应知识见 [reference/project-blog-map.html](reference/project-blog-map.html)。

**待决策（延后到第 4–5 阶段，等用户具备 React 基础后一起定）**：后端用
Java Spring Boot（用户的强项）还是 TS 全栈（better-auth 原生契合、SEO 一体化）。
在此之前，所有 CSS/JS/TS 课程与该决策无关。

## Constraints
- 用户偏好：全程用 Java/JVM/Spring 类比来强化记忆（核心教学手段）。
- 沟通语言：中文（代码与标识符用英文）。
- 框架顺序：先 React 深入，再 Vue 对照学习。
- CSS：系统补基础（不是「够用就行」），但控制在合理投入内。
- 节奏：每节课短、可快速完成、带一个可感知的小胜利；遵循可取难度（检索练习、间隔、交错）。
- 有付费课程 docs.3rcd.com 作交叉参考，但以本工作区的教学路线为准。

## Out of scope（当前阶段不追）
- Node.js 后端 / SSR 深入的内部机制 —— 等前端基础与一个框架打通后再说（SEO 触发时会重新评估）。
- 移动端原生（React Native 等）。
- 前端性能极致优化、WebGL、可视化等专项。
- Vue2 的历史包袱细节（仅作背景了解，主攻 Vue3）。
